Flame Admin - Documentation

Flame Admin is certainly unique. It is the result of months of hard work, thought, and careful tweaking so the end result is a beautiful interface with a powerful jQuery engine.

Still developing, Flame Admin will continue to be refined and added to, making it better with each update. You should periodically check your Themeforest profile "Downloads" section to see if there are any available updates.
Also, feedback and suggestions are welcome and will play a big part in shaping Flame Admin in the future. You can leave them as a comment on the item's page, or send me an email through my profile page.

To never see the thank you message again, remove <div id="modal-ov"> just before the wrapper closing tag.

You must have a local server like WAMP set up for LESS to work. If you don't have one, you can only view the production version of the site with compiled CSS.

I strongly recommend inspecting Flame Admin with your browser's Developer Tools as you're reading the documentation, to better understand its structure and code, and get a visual representation.

File structure

    • jquery-ui.css - jQuery UI stylesheet, currently only required by elFinder on the extras page
  • img/ - the image directory, of course
  • js/ - jQuery files and plugins (detailed file contents in the jQuery section)
    • cleditor.js - WYSIWYG text editor plugin
    • datatables.js - DataTables plugin
    • elfinder.js - elFinder file manager plugin
    • flot.js - Flot chart plotting plugin
    • functions.js - global functions and lightweight plugins
    • inputs.js - input functions like checkbox, form validation, tooltips, notifications and more
    • jquery.js, jquery-ui.js - jQuery and jQuery UI files
    • less.js - LESS parser
    • minicolors.js - colorpicker plugin
    • snippet.js - Snippet syntax highlighter (only used in documentation)
  • res/ - resources like fonts, the PHP files and upload directory for elFinder
  • 403, 404, 503.html - error pages, the bad news givers of the internet
  • analytics.html - lines, bars, pies, donuts (not the delicious kind, though)
  • dashboard.html - the homepage, the first page you see after logging in or hacking an account
  • extras.html - extras page, growl alerts, typography, data tables, file manager
  • grid.html - grid demo - sizes, combinations, nesting
  • inputs.html - the inputs page - buttons, selects, tooltips, forms and various fields
  • ui.html - UI elements page - accordion, sortable boxes, notifications and more
  • index, login-alt, login-sg.html - the login pages (user selection, alternative and single user)
  • recoverpass.html - password recovery page, for when the inevitable happens

The smaller plugin stylesheets have been included in ui.less, minified plugins have been included in functions.js and inputs.js. Larger plugins and stylesheets are standalone.

Basic page structure

<div id="wrapper"> - wraps all the page's content, fades in on page load.

<div id="nav"> - navigation menu, <div id="wrapper" class="hz-nav"> will make the navigation horizontal. 

        <div id="nav">
              <ul>
                      <li class="active"><span class="icon">Icon goes here</span>This is an active menu item</li>
                      <li><a href="link to page"></a><span class="icon">Icon</span>This is a basic menu item</li>
              </ul>
              <br class="clear">
        </div>

<div id="content"> - the actual content goes here, and the grid is calculated based on its width. It often has a page class, so content on that page can be easily arranged or styled (helpful for responsive code too).

<div id="modal-ov"> - the modal window container and overlay. Visible with class show.

</div> - end wrapper.

Boxes

Also known as content holders, panels, or "innard restrictors", boxes are containers for elements, text or what have you. There are 2 main types: simple, like this one and with header.

        <div class="box">I'm a simple box</div>
Box with header
            <div class="box">
                <div class="box-ttl">I'm a header</div>
                <div class="box-body">
                    And I'm the body!
                </div>
            </div>

Adding class coll to a box will make it collapsible. Adding class collapsed will make it collapsed by default.

Collapsible box, collapsed by default
And now it's not collapsed anymore.

To add an icon to a box header, just append <span class="icon">icon character here</span> to it. The same for buttons and button groups (code in the "Inputs" section).

If you want the icon on the left, add class icon-l to the header.

Content navigation

The tab-like content navigation is useful when you need to display a lot of information in a restricted space, or you just want to have tabs because you are an adult and can do whatever you want.

Content is transitioned vertically or horizontally with a sliding animation and fade in/out.
The default transition direction is horizontal, to make it vertical add class updown to the nav body. The selected tab gets class sel and its corresponding content gets class show. 

        <div class="box nav-box">
            <ul class="nav">
                <li data-nav="#tab1" class="sel">Tab 1</li>
                <li data-nav="#tab2">Tab 2</li>
            </ul>
            <div class="nav-body">
                <div id="tab1" class="nav-item show">Tab 1 content.</div>
                <div id="tab2" class="nav-item">Tab 2 content.</div>
            </div>
        </div>

The tab menu can be positioned on the top (default), left, right or bottom of the box or container. To set the menu position, add one of the classes below to the nav box:

  • nav-l - positions the menu to the left.
  • nav-r - positions the menu to the right.
  • nav-btm - positions the menu to the bottom. Because the menu is not absolutely positioned, it needs to be moved after the nav body in the markup.
  • $.fn.contNav(); - included in $.fn.loadfns();

Modal dialogs

A modal dialog's structure is similar to the box. The code below is for a modal with header, body and navigation menu, like the one you can see on every page if you click Settings or User CP.

        <div id="modal-ov">
            <div class="modal" id="mod-id">
                <div class="mod-ttl"><h2>MODAL TITLE</h2></div>
                <div class="mod-body">
                    <div id="tab1" class="nav-item show">Tab 1 content</div>
                    <div id="tab2" class="nav-item">Tab 2 content</div>
                </div>
                <div class="mod-act">
                    <ul class="mod-nav">
                        <li class="sel" data-nav="#tab1"><span class="icon">ICON</span></li>
                        <li data-nav="#tab2"><span class="icon">ICON</span></li>
                    </ul>
                    <button class="orange close">Close</button>
               </div>
             </div>
        </div>

To show a modal on click, assign an id to the element you want to trigger it and add a data-modal attribute with the id of the target modal. Then, call the modal function with $('#trigger-id').modal();.

If the modal's content is tabbed and you want the trigger to show a specific tab, also add the tab id in the data-modal attribute. 

<button id="trigger-id" data-modal="#mod-id #tab-id">Show modal</button>

When a modal is triggered, both the modal container (#modal-ov) and the triggered modal get class show. Clicking on the modal overlay or a button with class close will hide the modal.

Icons

To display an icon, just add class icon to the container and insert the character corresponding to the icon you want inside. To give it an inset appearance and dark color add class dark.

For arrows use class arrow instead.

You can see the icon and arrow character map in "Icons.jpg".

To make your own icon font, go to IcoMoon and choose from their huge collection of free icons. 

        <span class="icon">X</span>

Other elements

Tooltips

        <div class="tooltip">Tooltip text <span class="arrow">arrow</span></div>

Tooltips can be attached to any element, it just needs class ttip and a title or data-ttip="What you want the tooltip to say" attribute.

Options can be set with the data-ttip-opt attribute, get added to the tooltip element as classes, and can be the following:

  • top - positions tooltip to the top of the element (default).
  • left - positions tooltip to the left of the element.
  • right - positions tooltip to the right of the element.
  • bottom - positions tooltip to the bottom of the element.
  • dark - black tooltip instead of gray.

jQuery UI's "position" function is used to perfectly align the tooltip to the element, so jquery-ui.js must be included on pages with tooltips.

Notifications

        <div class="notif">
            Notification text <span class="icon">Icon</span>
            <p class="nt-det">Notification details</p>
        </div>

If a notification contains details (.nt-det) it will expand on first click and close on second. If it doesn't contain details, it will close on first click.

  • hide - hides notification.
  • auto - sets height to auto.
  • no-icon - removes extra padding on the right.
  • $.fn.notif(); - included in $.fn.loadfns();

Growl notifications

The growl notifications use toastr. Simple and easy to use, you can create a growl style notification with: 

        toastr.success('Success growl');

There are 4 types of growls: success, error, warning and info. You can also give it a title, and set options like position and timeout. The code below demonstrates how to show a growl when a button is clicked: 

        $('#btn').click( function() {
            toastr.options.positionClass = 'toast-top-left';
            toastr.error('Shutting down in... now' , 'Error');
        });

The position options are:

toast-top-right (default), toast-top-left, toast-bottom-right, toast-bottom-left, toast-top-full and toast-bottom-full.

Sliders

The jQuery UI slider code was modified to keep the handle on the track on line 10309.

        <div id="slider" class="slider orange"></div>

To enable the value/grade display, add class grade and call$('#slider').gradeslider();.

On horizontal sliders the grade is positioned on the bottom of the slider, adding class grade-top will move it to the top. On vertical sliders the grade is on the left by default, adding class grade-right will move it to the right.

The grid

Flame's grid is designed to be intuitive, without the need for a lot of markup and classes. Most of the time, all your element needs is a class of g1 to g16. Using CSS calc() enables grid elements to have exact margins in px, not percentages.

Boxes have a 20px margin on the right and bottom, other elements have 10px. You can use the helper class cont for containers inside boxes.

In the illustration below, you can see the container padding (green) and its elements' margin (orange). Because of this structure, there is no need for last classes unless the container also has padding on the right.

If your browser does not support CSS calc(), the grid will fall back to a percentage only grid. The wrapper has to have class altgrd for grid elements to have their margins set to percentages instead of pixels, and it gets added on load in functions.js. A percentage only grid is available in the Extended compatibility folder, which must be used with Safari 6, because it crashes with the calc() grid.

Buttons

Buttons and various other interface elements come in 7 colors: gray, red, blue, green, orange, purple and black.



There are 4 button sizes: btn-s small (24px), normal (26px), btn-m medium (34px) and btn-l large (38px).

To add an icon to a button, add class has-icon to it and append the icon you want. Class has-icon-r will move the icon to the right. Class icon2 instead of icon will pull the icon down by 2px.

For buttons that only contain the icon and no text, add class only-icon instead. If the icon is out of alignment, make a <div class="button"> instead of the button element, which ignores the line-height.
You can adjust the icon using the font-size and line-height helper classes (detailed in the CSS section). Hover over the blue icon to see the code.

Adding class toggle will make the button toggleable. Adding class pressed will activate the pressed state.

Button groups

Button groups are very easy to create, just make a div with class btn-group and append the buttons you want grouped to it.

Adding class btn-radio to a button group will "radio" the buttons. 

        <div class="btn-group">
            <button class="btn-m">Previous</button>
            <button class="btn-m">Next</button>
        </div>
        <div class="btn-group btn-radio">
            <button class="btn-m">This</button>
            <button class="btn-m">No, this</button>
        </div>

Input groups

You can add buttons to the left and/or right of a text input using an input group. jQuery is used to calculate the width of the input, so $.fn.inputgrp(); must be called inside $.fn.loadfns() (details in the jQuery section). 

        <div class="input-group">
            <button class="black input-comp">Left Button</button>
            <input type="text" value="The input">
            <button class="blue input-comp">Right Button</button>
            <button class="green input-comp">RB 2</button>
            <button class="input-comp">RB 3</button>
        </div>

Drops

There are 2 types of drops: single button or split button.

        <div class="drop single">
            <button class="btn-m black">Dropdown <span class="arrow">&</span></button>
            <ul>
                <li>Option 1</li>
                <li>Option 2</li>
            </ul>
        </div>
  • Option 1
  • Option 2

        <div class="drop">
            <button class="btn-m black">Dropdown</button>
            <button class="btn-m black arrow">&</button>
            <ul>...</ul>
        </div>
  • Option 1
  • Option 2
  • al-r - aligns the drop menu to the right of the button
  • arrow-l - moves arrow to the left
  • drop-l - sets drop direction to the left
  • drop-r - sets drop direction to the right
  • drop-u - sets drop direction upward
  • $.fn.btnDrop(); - included in $.fn.inputs();

Direction classes can be combined for more drop directions. For instance, drop-l drop-u will set the drop direction up and to the left.

Select

        <select>
            <option>Select</option>
            <option>An</option>
            <option>Option</option>
        </select>

Selects are transformed into the following HTML structure, to allow full customization:

        <div class="drop select">
            <select style="display: none;">...</select>
            <ul>
                <li class="sel">Select</li>
                <li>An</li>
                <li>Option</li>
            </ul>
            <span class="opt-sel">Select</span>
            <span class="arrow">&</span>
        </div>

<span class="opt-sel"> displays the selected option's value. If no option is selected, it gets the value of the first option.

When an option is selected, it gets the selected attribute added and its corresponding <li> gets class sel.

  • multiple - allows for multiple options to be selected, listed as tags
  • no-arrow - the arrow will not be added to the HTML
  • has-delete - adds clear button for resetting the select
  • disabled - disables transformed select, gets added if the select has attribute 'disabled'.
  • $.fn.makeSelect(); - transforms selects into the custom HTML structure, included in $.fn.loadfns(); (runs on load).
  • $.fn.flselect(); - provides functionality for transformed selects, called by $.fn.makeSelect(); , included in $.fn.inputs();

Checkboxes

        <input type="checkbox" class="chbox-mini">
        <input type="checkbox" class="chbox">
Like selects, checkboxes get transformed into the following HTML structure: 
          <div class="checkbox-cont unchecked chbox">
              <input type="checkbox" class="chbox" style="display: none;">
              <div class="on"><span>=</span></div>
              <div class="off"><span>X</span></div>
              <div class="check-mask"></div>
          </div>
Mini checkboxes are turned into: 
          <div class="checkbox-cont unchecked chbox-mini">
              <input type="checkbox" class="chbox-mini" style="display: none;">
              <div class="on"><span>=</span></div>
          </div>
  • disabled - disables transformed checkbox, gets added if the checkbox has attribute 'disabled'.
  • light-bg - changes box-shadows and makes the checkbox inset, for light backgrounds.
  • $('.chbox, .chbox-mini').flcheck(); - included in $.fn.loadfns();

Form validation

Flame's validation code is based on jQuery Tiny Validation, but most of the code has been written from scratch. The inputs that can be validated as required are text inputs, selects, checkboxes and spinners. Text input validation options include e-mail, card number or matching fields (for passwords, etc), and can be set a minimum number of characters with the data-min="x" attribute. Each input type has a custom error message, and text inputs have one for each option.

To validate a form, assign an id to it and add one of the classes below to the inputs in the form you want validated. Finally, call the jQuery validate function $('#form-id').validate();.

  • required - makes input required, form will not submit until it is valid.
  • email - requires a valid e-mail address format (xyz@xyz.xyz) to be entered.
  • cardnum - requires a valid credit card number format to be entered.
  • match - requires two inputs to match (each must have this class, and a data-match="same-for-every-input-in-set" attribute.

CSS

This section will explain the CSS, the stuff that gives Flame its darn good looks and animates most of it.
Each file will have the classes and IDs that need explaining listed below, and a description. The CSS code is organized by elements and each part is labeled, so finding and customizing what you need should be a piece of cake.
LESS makes the CSS more easily manageable, and it has other perks too. If you are unfamilliar with it, I recommend taking a look at their website, to get the basics of it.

To compile LESS files, you will need a compiling tool. WinLESS is one of them, but more can be easily found, just take your pick.

The compiled "style.css" file will contain the global and ui files as well. You can either delete the code until the /* FLAME STYLE.LESS */ comment and make separate files with @import tags (be sure to change the extension to .css instead of .less), or keep it that way and not import the global and ui stylesheet.

jQuery

functions.js

  • $.fn.togglefn() - toggles a class on the target on click. Trigger needs a data-tgcls="#target class" attribute. Automatically called on elements with class tgcls in $.fn.loadfns();
  • $.fn.contNav() - the content navigation function. If the element contains .nav-act, the wizard steps function is called to disable previous/next buttons when first/last tab is reached. Called on load in $.fn.loadfns();
  • $.fn.boxColl() - the box collapse function. The box height is added as a data-height attribute for easy access. Called on load in $.fn.loadfns();
  • $.fn.flaccordion() - the custom accordion function. Simpler than the jQuery UI one, and animates smoothly, without shaking. Called like: $('#accordion').flaccordion();
  • $.fn.loadfns() - the page load and global calls function. Also contains the function for absolutely positioning the navigation when the window height is below 674px and for showing the minified nav on click. If a page has functions that must be called on load, include them in the parameter as a function.
      $.fn.loadfns( function() {
          $.fn.calledOnLoad();
          $('.on-load').iAmCalled();
      };

inputs.js

  • $.fn.flcheck() - the checkbox function, included in $.fn.loadfns();
  • $.fn.loginNotif() - function for the login pages notifications
  • $.fn.notif() - notification function, included in $.fn.loadfns();
  • $.fn.scrollspy() - scrollspy function, as seen on the inputs and documentation pages.
  • $.fn.tooltip() - tooltip function, automatically called on elements with class ttip in $.fn.loadfns();
  • $.fn.btnDrop() - drop function for selects and drops, included in $.fn.inputs();

  • $.fn.validate() - function for validating forms. Has a function for each input type
  • $.fn.validateLogin() - function for validating the login pages inputs
  • $.fn.numberInp() - function for number only inputs (class number) and spinners (class spinner), included in $.fn.inputs();
  • $.fn.inputgrp() - function for input groups, should be called on load to properly detect the width
  • $.fn.gradeslider() - function for sliders with value/grade display
